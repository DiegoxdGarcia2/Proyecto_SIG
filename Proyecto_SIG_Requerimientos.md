Propuesta de Desarrollo SIG de una aplicación de monitoreo infantil
Por Ing Franklin Calderon Flores
MATERIA: INF442-SA (2025)
1. Resumen ejecutivo 
•	Desarrollo de una aplicación SIG para el monitoreo de niños prescolares. El sistema debe monitorear la localización del niño y detectar si el niño salió de un área definida como ser su centro educativo (Kinder) mediante el análisis espacial de información vectorial (Shapes), proporcionando una alerta en el celular de la madre o tutor del niño,  cuando el niño sale del área de monitoreo y así poder tomar algunas decisiones o acciones de búsqueda tempranas para evitar riesgos como, perdida del niño, accidentes o secuestros en casos extremos. 
2. Introducción y contexto del problema
•	La falta de una aplicación de monitoreo en tiempo real para detectar si el niño salió de su centro educativo (Kinder) mediante el análisis espacial de información vectorial, dificulta poder tomas decisiones tempranas a perdida del niño, accidentes o secuestros en la ciudad de Santa Cruz de la Sierra. Los controles manuales de monitoreo de un niño en su centro de educación son muy laboriosos para la madre o tutores y además son costosas y lentas, lo que provoca una respuesta tardía ante incidentes que puedan ocurrir al niño. También los SIG tienen herramientas óptimas para realizar operaciones espaciales que otros sistemas estándares no realizan.

3. Objetivos del proyecto
Define los objetivos de manera específica, medible, alcanzable, relevante y con un tiempo determinado (SMART). 
•	Objetivo general: Diseñar e implementar un sistema de información geográfica que permita el monitoreo en tiempo real de la posición de un niño prescolar y emitir una alerta en caso necesario.
•	Objetivos específicos:
-	Crear una base de datos geoespacial, datos de áreas o unidades educativas (Poligonos).
-	Desarrollar una aplicación escritorio, web o móvil para la visualización de la información y la generación de alerta.
-	Capacitar al personal encargado en el uso y mantenimiento de la aplicación.
4. Metodología
Detalla las fases y los métodos que se usarán para desarrollar el proyecto. 
•	Fase 1: Recopilación y preparación de datos: Obtención o levantamiento de datos en campo con GPS y/o digitalización de mapas existentes.
•	Fase 2: Diseño de la base de datos: Creación de un modelo de datos que integre capas de información geográfica (vectorial) y datos alfanuméricos.
•	Fase 3: Desarrollo del sistema: Implementación de la base de datos en un software SIG (por ejemplo, QGIS, ArcGIS) y desarrollo de la aplicación.
•	Fase 4: Análisis espacial: Realización de análisis de espacial para detectar anomalías en la posición geográfica del niño para emitir la alerta correspondiente 
5. Alcance del proyecto
Especifica claramente qué se incluye y qué no en el proyecto. 
•	Incluye: 
o	Diseño de la base de datos espacial y alfanumerico, desarrollo de la aplicación, y propuesta de soporte técnico inicial.
o	Adquisición de equipos de hardware propuestos para los usuarios.
o	 Adquisición de equipos de red o de comunicación
6. Recursos (hardware, software y personal)
Identifica los recursos tecnológicos y humanos necesarios para llevar a cabo el proyecto. 
•	Hardware: 
o	Servidor para las bases de datos, 
o	Computadoras o celulares.
o	Equipos de red
•	Software: 
o	Software SIG (por ejemplo: Arcgis, librerías de SIG para trabajar en entornos de desarrollo .NET u otros ) 
o	Sistema de gestión de bases de datos.
•	Personal: Especialistas en SIG, desarrolladores, cartógrafos, expertos en redes y telecomunicaciones. 
7. Conclusión
•	La implementación de esta aplicación SIG proporcionará una herramienta eficaz para la gestión de seguridad de niños o posiblemente de personas vulnerables, reduciendo la inseguridad y otros incidentes
